## Changelog

This release delivers hash-based config.json reconciliation for MCP tool groups, business units, and access profiles; a full gRPC-based cluster rewrite for improved node-to-node communication; and broad SCIM/SSO hardening including Azure credential injection via env vars, Google Directory ADC inheritance, and strengthened SCIM provider verify checks ‚Äî built on OSS base `transports/v1.5.0-prerelease6`.

## ‚ú® Features

### Config Reconciliation

- **MCP Tool Group Reconciliation** ‚Äî MCP tool groups are now reconcilable from `config.json` with hash-based sync, preventing unnecessary MCP client restarts on config reload; `tool_sync_interval` accepts Go duration strings
- **Business Unit Reconciliation** ‚Äî Business units can be seeded and reconciled declaratively from `config.json` with hash-based sync; added `business_units`, `team_id`, `calendar_aligned`, and `virtual_key_count` to governance schema and Helm
- **Access Profile Reconciliation** ‚Äî Access profiles are now reconcilable from `config.json` at deploy time, including provider restrictions, model allowlists, budgets, rate limits, and MCP server/tool controls
- **Guardrail Rule Query Field** ‚Äî Added `query` field support to guardrail rules config and reconciliation pipeline

### Cluster & Infrastructure

- **gRPC-based Custom Cluster** ‚Äî Full gRPC-based cluster implementation for enterprise-grade node-to-node communication with improved reliability
- **Governance Config Sync** ‚Äî Model configs and provider governance bindings now sync from `config.json` to the database at startup
- **Log Cluster Broadcasting Removed** ‚Äî Removed log cluster broadcasting and `EntityTypeLog` message handling to reduce cluster gossip overhead

### Security & SCIM

- **SCIM Provider Verify Improvements** ‚Äî Strengthened verify checks across SCIM providers for more reliable directory sync validation
- **Azure Credentials via Env Vars** ‚Äî Azure configuration fields now resolved from environment variables; TLS and proxy config fields (`url`, `username`, `password`, `ca_cert_pem`) also accept `env.VAR_NAME` for secure secret injection
- **Deny-by-Default Virtual Keys** ‚Äî Virtual key provider and MCP configs block all access when empty; automatic migration backfills existing keys to preserve existing behavior
- **`provider_key_name` Alias** ‚Äî Human-readable alias for routing targets and pricing overrides, resolved to `key_id` at config load time

### Enterprise Helm

- **Enterprise Helm Overlays** ‚Äî Suite of composable Helm overlays for guardrails, org governance, access profiles, customer budgets, teams, multi-customer governance, and SCIM/SSO
- **Semantic Cache Helm Layers** ‚Äî Added `values-semantic-search-redis.yaml` and `values-semantic-search-weaviate.yaml` for Redis and Weaviate-backed semantic cache deployments
- **Key IDs in Helm** ‚Äî `key_ids` is now the preferred field for pinning provider keys in Helm virtual key configurations, aligning Helm with `config.json` schema

### AI Provider Support

- **Claude Opus 4.7** ‚Äî Full support including adaptive thinking, task-budgets beta header, `display` parameter handling, and "xhigh" effort mapping
- **Anthropic Structured Outputs** ‚Äî `response_format` and JSON-schema/JSON-object support for Anthropic chat completions and Responses API
- **Anthropic Server Tools** ‚Äî End-to-end support for server-side tools: web search, code execution, and computer use containers
- **MCP Tool Annotations** ‚Äî Preserve tool annotations (`title`, `readOnly`, `destructive`, `idempotent`, `openWorld`) in bidirectional conversion so agents can reason about tool behavior

### Routing & MCP

- **Auto-fill Incoming Model for Fallbacks** ‚Äî Routing rule fallback entries can omit the model; the incoming request model is substituted automatically at runtime
- **Namespace Tool Type** ‚Äî Namespace tool container type in Responses API; non-OpenAI providers receive automatically flattened tool lists
- **MCP Tool Groups Governance Scoping** ‚Äî `tool_groups` config supports governance scoping by virtual key, team, customer, user, provider, and API key

### Pricing & Observability

- **Cache Creation Pricing** ‚Äî Cache creation details for Claude models with 5-minute and 1-hour TTL pricing tiers
- **Period Parameter in Dashboard/Logs APIs** ‚Äî Added `period` param to support relative time range queries on dashboard and logs endpoints
- **Team Budgets** ‚Äî New team budget system with per-team spending tracking and atomic ratelimit updates
- **Single Log Export** ‚Äî Export individual log entries from the logs view and MCP logs sheet

### UI

- **User Ranking Graph** ‚Äî Updated user ranking graph layout and visualization

## üêû Fixed

### Enterprise

- **Google Directory API Inherited ADC** ‚Äî Support inherited Application Default Credentials for Google Directory API so deployments using Workload Identity / pod service accounts work without explicit key files
- **Guardrail Output Rule Nil Check** ‚Äî Early exit if response is nil for output rules in guardrails to prevent nil pointer panics
- **Access Profile Live Limits** ‚Äî Enforced live access profile limits and preserved usage on propagation to ensure rate limits are respected across reloads
- **Access Profile UI Layout** ‚Äî Fixed access profile sheet width and padding for better layout

### OSS (prerelease5 + prerelease6)

- **WebSocket /responses Reliability** ‚Äî Fixed upstream handshake diagnostics, proper error capture, and WebSocket connection lifecycle in the native `/responses` path
- **Routing Rule Query Normalization** ‚Äî Normalized `query` field to valid `RuleGroupType` and tightened schema validation
- **Budget and Team Co-creation** ‚Äî Fixed creation of budgets and teams in the same request
- **Provider Reload** ‚Äî Fixed keyless provider status updates during config reload; provider runtime now reloads correctly after key creation
- **OTel Metrics** ‚Äî Fixed OpenTelemetry metrics pipeline not working (thanks [@tcx4c70](https://github.com/tcx4c70)!)
- **OTel Export** ‚Äî Fixed OTEL exporting to correctly show input and output messages; OTel plugin now defaults `insecure` to `true` when omitted for HTTP collectors
- **Multipart File Uploads** ‚Äî Write multipart metadata before file content to fix upload ordering
- **Env Var Redacted Check** ‚Äî Added missing redacted check for env var values
- **PydanticAI Null Text Fields** ‚Äî Normalized null text content in PydanticAI stream response chunks
- **Provider Queue Shutdown Panic** ‚Äî Eliminated `send on closed channel` panics in provider queue shutdown
- **Bedrock Streaming** ‚Äî Emit `message_stop` event for Anthropic invoke stream; case-insensitive `anthropic-beta` header merging
- **Bedrock Tool Images** ‚Äî Preserve image content blocks in tool results when converting to Bedrock Converse API
- **Gemini Tool Outputs** ‚Äî Handle content block tool outputs in Responses API path for `function_call_output` messages
- **Gemini Thinking Level** ‚Äî Preserved `thinkingLevel` parameters across round-trip conversions; corrected finish reason mapping
- **vLLM Token Usage** ‚Äî `delta.content=""` treated same as `nil` in streaming; restores token usage attribution in logs and UI
- **Anthropic WebSearch** ‚Äî Removed Claude Code user agent restriction so WebSearch tool arguments flow for all clients
- **Responses Streaming Errors** ‚Äî Capture errors mid-stream in the Responses API so clients see failures instead of silent termination
- **Anthropic Request Fallbacks** ‚Äî Dropped fallback fields from outgoing Anthropic requests to avoid schema validation errors
- **Custom Providers** ‚Äî Allow custom providers without a list-models endpoint to accept any model
- **Async Context Propagation** ‚Äî Preserve context values in async requests so downstream handlers retain request-scoped data
- **Access Profile Rate Limits** ‚Äî Rate limit counters for access profiles now persisted correctly to the database
- **Fallback Stream State** ‚Äî Clear `BifrostContextKeyStreamEndIndicator` before fallback requests so stale streaming state doesn't carry into retries
